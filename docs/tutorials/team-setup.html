Team Setup Guide

Onboarding Your Team

Rendro uses Google OAuth for authentication, so team members can sign in with their existing Google accounts. Each member's organization is determined by their email domain.

For teams using Gmail, the org slug is gmail. For Google Workspace domains, it's your domain name (e.g., acme-corp for @acme-corp.com).

Setting Up Permissions

API keys are per-organization. Share the API key with team members who need to push docs. For CI/CD, store the key as a repository secret.

Currently, all org members have write access. Future versions will support fine-grained permissions and multiple API keys per org.

Best Practices

Review Before Push — Use pull requests to review doc changes before merging to main. The CI workflow triggers on push to main, so reviewed changes go live automatically.

Organize by Topic — Use nested folders to mirror your documentation structure. The sidebar tree preserves your folder hierarchy.

Link Between Pages — Use relative links (<a href="../api/overview.html">) for cross-references. The tree UI tracks navigation and highlights the current page.
