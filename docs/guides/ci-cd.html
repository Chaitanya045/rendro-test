CI/CD Setup

GitHub Actions

The recommended CI/CD setup uses GitHub Actions with a simple curl-based script. Store your Rendro API key as a repository secret (RENDRO_API_KEY) and create a workflow that runs on push to main.

name: Sync Docs
on:
  push:
    branches: [main]
jobs:
  sync:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Sync
        env:
          RENDRO_API_KEY: ${{ secrets.RENDRO_API_KEY }}
        run: |
          # Use the Rendro sync script
          npx rendro push --source ./docs --org my-org

GitLab CI

For GitLab, use a similar approach with the GitLab CI pipeline. Store the API key as a CI/CD variable and add a job that runs on merge to main.

Manual Sync

For teams without CI/CD, the CLI can be run manually or via a pre-commit hook. The hash-based diffing ensures manual syncs are fast, even for large doc sets.
