Sync API Endpoints

POST /api/sync/upload

Upload a file to blob storage. The file key must start with the org prefix. Content is stored as-is with the specified content type.

curl -X POST https://rendro.app/api/sync/upload \
  -H "Authorization: Bearer rendro_xxx" \
  -H "Content-Type: application/json" \
  -d '{"key": "gmail/index.html", "content": "
Hello
"}'

GET /api/sync/check

Check if a file exists on the server and optionally compare its hash. Returns exists, etag, and match fields.

curl "https://rendro.app/api/sync/check?key=gmail/index.html&hash=abc123" \
  -H "Authorization: Bearer rendro_xxx"

GET /api/sync/list

List all files in the organization, filtered by soft-delete status.

curl https://rendro.app/api/sync/list \
  -H "Authorization: Bearer rendro_xxx"

DELETE /api/sync/delete

Soft-delete a file. The file is hidden from listings but remains accessible via direct URL.

curl -X DELETE "https://rendro.app/api/sync/delete?key=gmail/old.html" \
  -H "Authorization: Bearer rendro_xxx"
