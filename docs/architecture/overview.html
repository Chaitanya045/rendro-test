Architecture Overview

System Design

Rendro follows a serverless architecture built on Cloudflare Workers. The system consists of three main components: a Cloudflare Worker for the web interface, Convex for authentication and data storage, and R2 for document blob storage.

The CLI communicates with the Worker via REST API endpoints under /api/sync/*. Authentication uses per-organization API keys stored as SHA-256 hashes in Convex.

Data Flow

Write Path: CLI → Worker API → R2 (blob storage). Files are uploaded with hash-based deduplication — unchanged files are skipped.

Read Path: Browser → Worker → R2 (stream file). The Worker injects the commentor widget before serving HTML to enable inline comments.

Auth Path: Browser → Worker → Convex (better-auth). OAuth flow is proxied through the Worker to Convex HTTP actions.
