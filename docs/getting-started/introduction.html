Introduction to Rendro

What is Rendro?

Rendro is a documentation hosting platform that turns static HTML files into a live, searchable knowledge base. Write docs as plain HTML, push via CLI, and Rendro serves them instantly with a VS Code-style sidebar tree, inline comments, and full-text search.

Unlike traditional documentation tools that require complex build pipelines or framework lock-in, Rendro works with raw HTML files — no JavaScript frameworks, no static site generators, no configuration files. Just write HTML, push, and read.

How It Works

Rendro follows a simple pipeline: Write → Push → Read. You author your documentation as HTML files in your repository's docs/ directory. The Rendro CLI scans your files, computes hashes, and uploads only changed files to blob storage. Your team accesses the live docs through the Rendro web interface.

The web interface provides a VS Code-style sidebar tree with lazy-loading folders, infinite scroll for large directories, sticky folder headers, and dark mode support. Inline comments allow team members to leave feedback directly on documentation.

# Install the CLI
npm install -g rendro

# Push your docs
rendro push --source ./docs --org my-org

Key Concepts

Organizations — Your docs are grouped under an organization slug, typically derived from your email domain. Each org has its own namespace in blob storage.

API Keys — Per-organization API keys authenticate CLI operations. Generate one from the web dashboard after creating your org.

Soft Delete — Removed files aren't permanently deleted. They're hidden from the tree listing but remain accessible via direct URL, providing a safety net for accidental deletions.
